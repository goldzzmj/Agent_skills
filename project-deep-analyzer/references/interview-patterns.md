# Interview Patterns Library

高阶面试问题模式库，用于生成具有区分度的技术面试问答。

---

## 问题类型分类

### Type 1: 原理深入型 (Why-Deep)

**特征**：考察对底层原理的理解深度

**问题模板**：
- "为什么 {组件} 使用 {机制} 而不是 {替代方案}？"
- "{算法} 中的 {参数/步骤} 有什么作用？去掉会怎样？"
- "为什么 {论文} 要这样设计 {模块}？背后的数学直觉是什么？"

**示例**：
```markdown
Q: 为什么 Transformer 使用 Pre-LN 而不是 Post-LN？

A: Post-LN（原始 Transformer）存在训练不稳定问题：
   - 残差连接后的梯度需要通过 LayerNorm
   - 容易出现梯度消失/爆炸
   Pre-LN（GPT-2/3 采用）优势：
   - 梯度直接通过残差连接传递
   - 训练更稳定，可使用更大学习率
   - 缺点是性能略有下降（~0.5%）
```

### Type 2: 对比分析型 (Compare)

**特征**：考察对多种方案的对比理解

**问题模板**：
- "{方案A} 和 {方案B} 的区别是什么？各有什么优缺点？"
- "在 {场景} 下，应该选择 {方案A} 还是 {方案B}？为什么？"
- "比较 {模型A} 和 {模型B} 在 {任务} 上的表现差异"

**示例**：
```markdown
Q: 比较 Flash Attention 和标准 Attention 的区别？

A: | 维度 | 标准 Attention | Flash Attention |
   |-----|---------------|-----------------|
   | 显存 | O(N²) | O(N) |
   | 速度 | 基准 | 2-4x 加速 |
   | 精度 | FP32/FP16 | 相同 |
   | 原理 | 完整存储注意力矩阵 | 分块计算 + 在线 softmax |
```

### Type 3: 场景应用型 (Scenario)

**特征**：考察实际应用能力

**问题模板**：
- "在 {生产环境/资源受限} 场景下，如何部署 {模型}？"
- "当遇到 {问题} 时，如何诊断和解决？"
- "如何优化 {模型} 的推理延迟/显存占用？"

**示例**：
```markdown
Q: 如何在 8GB 显存的 GPU 上部署 7B 参数的模型？

A: 1. 量化：INT8/INT4 量化，显存降低 2-4x
   2. 模型并行：将模型分片到多卡
   3. KV Cache 优化：PagedAttention、MQA/GQA
   4. Offloading：CPU-GPU 混合推理
   5. 动态批处理：提高吞吐量
```

### Type 4: 优化改进型 (Optimize)

**特征**：考察问题解决能力

**问题模板**：
- "如何提升 {模型} 的 {指标}？"
- "{模型} 在 {场景} 下表现不佳，如何改进？"
- "发现 {问题}，请给出优化方案"

**示例**：
```markdown
Q: 训练 Loss 震荡不收敛，可能的原因和解决方案？

A: 可能原因：
   1. 学习率过大 → 降低 LR 或使用 warmup
   2. Batch size 过小 → 增大或使用 gradient accumulation
   3. 数据问题 → 检查标签噪声、数据分布
   4. 梯度爆炸 → Gradient clipping
   5. 网络结构问题 → 检查残差连接、初始化
```

### Type 5: 权衡取舍型 (Trade-off)

**特征**：考察系统设计思维

**问题模板**：
- "如何平衡 {指标A} 和 {指标B}？"
- "在 {约束} 下，如何做出取舍？"
- "{方案} 的代价是什么？值得吗？"

**示例**：
```markdown
Q: 如何平衡模型精度和推理速度？

A: 1. 蒸馏：用大模型指导小模型，精度损失小
   2. 量化：INT8 精度损失 ~1%，速度提升 2x
   3. 剪枝：移除不重要的神经元/头
   4. 架构搜索：寻找更高效的架构
   5. 投机解码：小模型 draft，大模型 verify
```

## 答案结构模板

### 标准答案格式

```markdown
### Q: {问题}

**分类**：{类型}

**答案**：

1. **核心原理**（一句话概括）
   - ...

2. **详细解释**
   - 技术细节
   - 数学推导（如有）
   - 代码示例（可选）

3. **对比/权衡**
   - 与其他方案的对比
   - 优缺点分析

4. **实践建议**
   - 适用场景
   - 注意事项

**关键要点**：
- 要点 1
- 要点 2
- 要点 3

**延伸问题**：{更深层的问题}
```

## 问题难度分级

### L1 - 基础理解 (Junior)

- 能回答说明了解基本概念
- 示例："Transformer 的基本结构是什么？"

### L2 - 应用实践 (Mid-level)

- 能回答说明有实际项目经验
- 示例："如何解决训练中的梯度消失问题？"

### L3 - 深入原理 (Senior)

- 能回答说明对底层有深入理解
- 示例："解释 Flash Attention 的在线 softmax 算法？"

### L4 - 创新思考 (Staff/Principal)

- 能回答说明有系统设计和创新能力
- 示例："如果要支持 1M tokens 上下文，你会如何设计系统架构？"

## 生成策略

针对不同项目类型，选择合适的问题组合：

| 项目类型 | 推荐问题组合 |
|---------|------------|
| LLM 预训练 | 2x L3 + 2x L4 + 1x L2 |
| LLM 微调 | 2x L2 + 2x L3 + 1x L4 |
| 多模态 | 1x L2 + 2x L3 + 2x L4 |
| 推理优化 | 2x L3 + 3x L4 |
| CV 模型 | 2x L2 + 2x L3 + 1x L4 |

确保每个分析文档包含 5 个问题，覆盖至少 3 种类型。
