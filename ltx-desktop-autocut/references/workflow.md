# LTX Desktop 自动剪辑实践总结

## 已验证的经验

### 1. 自动化不应从“全 GUI 剪辑”开始

如果直接在 LTX 内逐图导入、逐图设时长、逐段卡点，自动化难度会快速失控。更稳的路线是：

- 外部先缩图
- 外部先拼音乐
- 外部先生成节奏明确的视频底稿
- LTX 只负责导入、可能的小修、导出

### 2. 中文路径对文件对话框不稳定

在实际操作中，带中文路径时，导入对话框容易：

- 路径识别失败
- 文件名输入后被错误解释
- OCR 返回大量乱码

最佳实践：

- 把需要导入的文件复制到如 `E:\Project\temp\ltxdrop\ready.mp4`
- 让 LTX 从这个目录导入

### 3. OCR 是辅助，不是唯一依据

OCR 可以稳定识别：

- 顶部菜单
- `Import Media`
- `Export`
- `Assets`
- `Timeline 1`

但在复杂文件选择器或缩略图列表中，OCR 不一定稳定。因此：

- 用 OCR 找按钮
- 用固定工作流减少需要 OCR 的步骤

### 4. 恢复窗口是必须动作

LTX 在最小化状态下，窗口坐标可能是负大值。任何点击前都应：

- `restore`
- `activate`
- 再重新读取窗口几何信息

## 婚礼/领证类视频推荐节奏

### 时长

- 推荐 180 秒左右

### 画面节奏

- 快节奏段：`0.68s ~ 0.95s`
- 中节奏段：`0.95s ~ 1.15s`
- 慢情绪段：`1.20s ~ 1.60s`

### 推荐结构

1. 开场亮相
2. 双人互动
3. 快切高潮
4. 情绪放缓
5. 尾奏收束

### 适合的镜头效果

- 轻微 push-in
- 轻微 zoom-out
- 短淡入淡出
- 少量硬切

## 推荐的自动化回退路径

### 回退 1：LTX 导入失败

- 把文件复制到 ASCII 路径
- 再次导入

### 回退 2：OCR 下载模型失败

- 设置代理后重试
- 仍失败时，只保留固定坐标方案

### 回退 3：复杂时间线操作失败

- 改为导入已拼好的 `ltx_ready_beatcut.mp4`
- 直接走导出流程
