# 日报生成 Skill

自动分析项目文件夹，读取脚本、数据、日志等信息，生成结构化的markdown日报。

## 使用场景

当用户需要：
- 生成项目日报
- 记录工作进展
- 汇报项目成果
- 创建周报/月报

## 工作流程

### 步骤1: 选择项目文件夹

首先询问用户要生成日报的项目文件夹路径。

**示例问题**:
```
请提供要生成日报的项目文件夹路径？
例如: path
```

### 步骤2: 探索项目结构

使用Glob和Grep工具探索项目目录结构，收集：
- Python脚本文件 (.py)
- 数据文件 (.xlsx, .csv, .json)
- 日志文件 (.log)
- 配置文件
- 文档文件

### 步骤3: 读取关键文件内容

读取以下文件内容：
1. **主要脚本文件** - 了解项目功能
2. **日志文件** - 提取执行信息、错误、统计
3. **数据文件** - 分析数据结构
4. **配置文件** - 了解项目配置
5. **现有日报** (如有) - 参考格式

## 数据提取规则

### 从日志文件提取

**时间信息**:
- 查找模式: `开始时间`、`结束时间`、`执行时间`
- 示例: `2026-02-26 15:30 - 16:32 (约62分钟)`

**统计信息**:
- 查找模式: `处理完成`、`成功`、`失败`、`总计`
- 示例: `成功 199/200`

**错误信息**:
- 查找模式: `ERROR`、`WARNING`、`失败`

### 从脚本文件提取

**功能描述**:
- 从docstring提取
- 从函数名推断

**关键参数**:
- 查找常量定义
- 查找配置变量

### 从数据文件提取

**数据结构**:
- 读取Excel/CSV列名
- 分析数据类型

**数据统计**:
- 行数统计
- 唯一值统计
- 分布分析

## 可视化条生成规则

百分比转可视化条:
```
0-5%:    █
5-10%:   ██
10-20%:  ████
20-30%:  ████████
30-50%:  ████████████████
50-70%:  ████████████████████████████
70-85%:  ████████████████████████████████████
85-100%: ████████████████████████████████████████████████
```

公式: `可视化条长度 = int(百分比 / 2) * '█'`

## 注意事项

1. **文件编码**: 读取文件时注意中文编码问题
2. **路径处理**: Windows路径使用原始字符串 `r'path'`
3. **数据安全**: 不泄露敏感信息（API密钥、密码等）
4. **日志格式**: 不同项目的日志格式可能不同，需适配
5. **时区处理**: 注意日志中的时区信息

## 输出示例

生成的日报文件应保存在项目根目录，文件名格式：
```
日报_2026-02-26.md
```
